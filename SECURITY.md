# Security Policy

The TrenchMap team takes security seriously. We appreciate your efforts to responsibly disclose your findings, and we will make every effort to acknowledge your contributions.

## Supported Versions

To ensure you are using the most secure version of TrenchMap, we only provide security updates for the latest stable release. Please ensure you are running the most current version of the project available in the `main` branch.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

**Please do not create public GitHub issues for security vulnerabilities.**

We take all security reports seriously. If you discover a security vulnerability, an exploit, a bug, or have any security concerns, please report it to us privately.

To report a vulnerability, please send a Direct Message (DM) on Twitter to:
**[https://x.com/naMqe7](https://x.com/naMqe7)**

Please include a detailed description of the issue, steps to reproduce it, and any other relevant information that can help us understand and address the vulnerability.

We will review all reports and do our best to address them promptly. We thank you for your help in keeping TrenchMap secure.
